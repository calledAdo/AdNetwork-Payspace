# Memory: Buyer Agent

Memory is stored as JSON files in `$AGENT_DIR/memory/`.

**Atomic writes (required):** Always write JSON memory files via a temp file and rename,
never write directly to the target path. This prevents the gateway from reading partial data:
```bash
TMP=$(mktemp "$TARGET.XXXXXX")
echo "$JSON" > "$TMP"
mv "$TMP" "$TARGET"
```
`mv` (rename) is POSIX-atomic — readers always see either the old or the new complete file.

## `placements.json` — one grouped record per placement lifecycle

Initialize to `[]`.

```json
[
  {
    "state": "streaming",

    "discovery": {
      "placement_id": "0xabc...:0",
      "placement_tx_hash": "0xabc...",
      "placement_index": 0,
      "seller_pubkey": "0x03...",
      "seller_blake160": "0x...",
      "seller_a2a_url": "https://block.example/a2a/agent_seller_xyz",
      "gateway_url": "https://block.example/a2a/agent_seller_xyz"
    },

    "slot_details": {
      "snippet_id": "snip_abc123",
      "page_url": "https://publisher.example/article-1",
      "dimensions": "728x90",
      "min_amount_per_1000": "1200",
      "ad_position": 0,
      "publication_mode": 1,
      "keyword_flags": "4",
      "policy_text": "No gambling",
      "metadata": {}
    },

    "conversation": {
      "correlation_id": "buyer_nonce_abc123",
      "context_id": "ctx-abc123"
    },

    "negotiation": {
      "offered_price_per_mille": "1000",
      "agreed_price_per_mille": "1100"
    },

    "delivery": {
      "booking_id": "book_123",
      "tracked_image_url": "https://backend.example/tracking/image?booking_id=book_123",
      "tracked_click_url": "https://backend.example/tracking/click?booking_id=book_123",
      "live_url": "https://publisher.example/article-1",
      "element_id": "snip_abc123"
    },

    "verification": {
      "last_verified_at": 1712000000,
      "verification_failures": 0,
      "last_result": "ok"
    },

    "payment_channel": {
      "channel_tx_hash": "0x...",
      "channel_index": 0,
      "channel_id": "0x...",
      "total_udt_locked": "1000000"
    },

    "payments": {
      "last_ticket_seller_claim": "4200",
      "last_ticket_timestamp": 1712000000,
      "last_ticket_sig": "0x..."
    },

    "tasks": {
      "campaign_handoff": {
        "task_id": "task-campaign-123",
        "state": "TASK_STATE_COMPLETED"
      },
      "channel_open_confirm": {
        "task_id": "task-booking-456",
        "state": "TASK_STATE_WORKING"
      },
      "coop_close": null
    },

    "manual_submission_deadline": null
  }
]
```

Notes:

- `discovery.placement_id` is derived from the BookingSpace outpoint: `tx_hash:index`
- `discovery.placement_id` is the current slot locator for fresh discovery and first seller contact
- `discovery.placement_id` is not guaranteed to remain stable after later on-chain slot mutations
- `conversation.correlation_id` is generated by the buyer before the first negotiation message
- `conversation.context_id` remains `null` until the seller assigns it
- once `conversation.context_id` exists, conversation continuity should use `context_id` rather than `placement_id`
- `slot_details` comes from the seller agent via `get-slot-details`, not from the backend
- buyer-side campaign performance should use `delivery.booking_id`
- payment/dispute identity should use `payment_channel.channel_id` and `payment_channel.channel_tx_hash`
- `tasks` keeps seller task ids and current task states together

Valid `state` values:
`negotiating` → `awaiting_publication` → `verifying` → `channel_open` →
`streaming` → `closing` → `closed` / `disputed`

## `tickets/{placement_id_underscore}.jsonl` — append-only payment ticket log

Example entry:

```json
{"seq":1,"sellerClaimUdt":"4200","ticketTimestamp":1712000000,"channelId":"0x...","signature":"0x...","sentAt":1712000001}
```

This file is used for:

- dispute fallback
- ACK timeout analysis
- payment audit trail
